Available states
----------------

.. contents::
    :local:

``netdata``
^^^^^^^

Installs netdata, manages configuration and starts service.

``netdata.install``
^^^^^^^^^^^^^

Installs netdata from git.

``netdata.config``
^^^^^^^^^^^

This state will configure the netdata service and has a dependency on ``netdata.install``
via include list.

``netdata.service``
^^^^^^^^^^^^^^^^^^^^

This state will start the netdata service and has a dependency on ``netdata.config``
via include list.

``netdata.clean``
^^^^^^^^^^^^^^^^^^

Uninstalls netdata.

Testing
-------

Linux testing is done with ``kitchen-salt``.

Requirements
^^^^^^^^^^^^

* Ruby
* Docker

.. code-block:: bash

   $ gem install bundler
   $ bundle install
   $ bin/kitchen test [platform]

Where ``[platform]`` is the platform name defined in ``kitchen.yml``,
e.g. ``debian-9-2019-2-py3``.

``bin/kitchen converge``
^^^^^^^^^^^^^^^^^^^^^^^^

Creates the docker instance and runs the ``template`` main state, ready for testing.

``bin/kitchen verify``
^^^^^^^^^^^^^^^^^^^^^^

Runs the ``inspec`` tests on the actual instance.

``bin/kitchen destroy``
^^^^^^^^^^^^^^^^^^^^^^^

Removes the docker instance.

``bin/kitchen test``
^^^^^^^^^^^^^^^^^^^^

Runs all of the stages above in one go: i.e. ``destroy`` + ``converge`` + ``verify`` + ``destroy``.

``bin/kitchen login``
^^^^^^^^^^^^^^^^^^^^^

Gives you SSH access to the instance for manual testing.
